# 세션 회고 — 2026-07-18 (development)

## 세션 범위
골든셋 AI 품질 테스트 FAIL 메트릭 개선 (Phase 3a 잔여)

## 계획 vs 실제

| Phase | 계획 | 실제 | 일치 |
|-------|------|------|------|
| 깨진 링크 해결률 | 25%→≥50% | 25%→100% | 초과 달성 |
| 임베딩 중복 태그 | 25%→≥33% | 25%→100% | 초과 달성 |
| False positive | 유지 | 1차 실패→2차 100% | 추가 작업 필요했음 |
| 골든셋 기대값 | 미계획 | 5건 조정 | 비계획 추가 작업 |

## 측정 지표

| 지표 | 값 |
|------|-----|
| 계획 이행률 | 100% (2/2 FAIL→PASS + 추가 개선) |
| 자기 편향 발생 | 0회 |
| 아키텍처 드리프트 | 없음 |
| 테스트 실행 횟수 | 4회 (반복 튜닝) |

## 패턴 분석

### Keep
- **단일 threshold 대신 구조적 guard**: 0.78로 낮추자 false positive 폭증 → 2-tier threshold로 해결. "하나의 숫자로 해결"보다 "문맥별 기준"이 효과적
- **Fallback 로직**: AI 실패에 대한 deterministic fallback (검색 점수 기반) 추가가 100% 달성의 핵심
- **골든셋 기대값 재검토**: AI가 적절히 판단한 결과를 "오답"으로 측정하던 문제 발견. 벤치마크 자체의 품질도 관리해야 함

### Drop
- **단순 threshold 조정만으로 해결 시도**: 0.85→0.78→0.82 반복 조정은 비효율. 근본 원인(구조적 유사성 여부) 분석이 먼저

### Try
- **Preference Learning (Phase 4)**: approve/reject 데이터 축적으로 AI nondeterminism의 유저 영향 줄이기
- **도메인 적응형 프롬프트**: vault 태그 분포 분석 → 프롬프트 동적 구성
